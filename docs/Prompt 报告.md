# Prompt 报告 — RAG 智能知识库问答系统

## 概述

本系统的 Prompt 管理遵循"配置化 + 数据库持久化"的设计原则，所有提示词通过 `PromptTemplate` 实体存储在数据库中，通过 `TemplateService` 进行管理和渲染，避免了代码中的硬编码。

---

## 版本记录

### v1.0 — 基础 RAG 模板（当前使用）

**模板ID**: `rag-default`

**模板内容**:
```
你是一个智能问答助手。请根据以下参考资料回答问题。

{{context}}

请基于以上资料，用中文简洁明了地回答问题。
如果资料中没有相关信息，请如实说明。
回答时标注引用来源编号，如 [1][2]。
```

**上下文变量注入**:
```
--- 参考资料 1 ---
来源：[文档标题]
[文档内容片段]

--- 参考资料 2 ---
来源：[文档标题]
[文档内容片段]
```

**迭代说明**:
- 第一版：仅包含"根据参考资料回答问题"的简单指令
- 第二版（当前）：增加了三个关键约束：
  1. 无资料时如实说明 → 减少幻觉
  2. 标注引用来源编号 → 提高可追溯性
  3. 用中文回答 → 明确语言要求

**效果评估**: 该模板适用于通用 RAG 问答场景，对多文档引用场景表现良好。

## Prompt 调用链路

```
用户提问 → EmbeddingService.search() 向量检索
         → 获取 Top-5 相关文档片段
         → buildSystemPrompt() 构建完整 Prompt
             → TemplateService.renderTemplateByName("rag-default")
             → 将文档片段注入 {{context}} 占位符
         → ChatModel.stream(Prompt) 调用 LLM
         → SSE 流式返回 token → 前端拼接展示
```

---

## 变量设计

| 变量名 | 类型 | 说明 | 来源 |
|--------|------|------|------|
| `{{context}}` | String | 检索到的相关文档片段集合 | EmbeddingService.search() |
| `{{query}}` | String | 用户原始问题（预留） | ChatRequest.message |

## 自定义模板

用户可以通过系统提供的"Prompt 模板管理"功能在运行时创建和修改模板：

**支持的特性**:
- CRUD 操作：创建、查看、更新、删除模板
- 分类管理：按场景（rag、general、translate 等）分类
- 变量注入：使用 `{{variableName}}` 语法定义占位符
- 使用统计：自动记录每个模板的调用次数
- 实时生效：修改模板内容后无需重启服务

**示例用途**:
1. **翻译模板**: "请将以下内容翻译成英文：\n\n{{context}}"
2. **总结模板**: "请对以下内容进行摘要总结，不超过 200 字：\n\n{{context}}"
3. **代码审查模板**: "请审查以下代码，指出潜在问题：\n\n{{context}}"

---

## 兜底策略

当数据库中的 `rag-default` 模板不存在时（首次启动场景），系统使用 `DataInitializer` 自动创建默认模板。如果模板渲染失败，ChatService 的 `buildSystemPrompt()` 方法会使用硬编码的兜底提示词，确保系统在任何情况下都能正常工作。

## 优化方向（待办）

1. **Query 重写**: 利用 LLM 将用户原始问题重写为更适合检索的表述
2. **Few-shot 示例**: 在模板中加入问答示例，提升回答格式一致性
3. **动态 Temperature**: 根据问题类型动态调整生成参数
4. **多轮对话上下文**: 将历史对话注入模板，支持追问场景
5. **模板版本控制**: 记录模板修改历史，支持回滚
