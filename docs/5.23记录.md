# 5.23 问题解决记录

## 项目概览

RAG 智能知识库问答系统，基于 Spring Boot 3.2.5 + Vue 3 + Ollama + pgvector。

## 已解决问题

### 1. 后端启动问题

- **端口 8080 被占用** — 多次出现旧进程残留导致 `WebServerException: Port 8080 was already in use`。通过 `taskkill /PID <pid> /F` 强制终止占用进程后恢复正常启动。
- **Maven 依赖冲突** — `spring-boot-starter-webflux` 与 `spring-boot-starter-web` 共存时 Tomcat 与 Netty 冲突，通过显式排除 WebFlux 的嵌入式服务器解决（保留 WebFlux 依赖用于 WebClient，但让 Tomcat 作为主容器）。
- **Spring AI BOM 版本** — 使用 `spring-ai-bom:1.0.0` 管理 AI 相关依赖版本，确保与 Spring Boot 3.2.5 兼容。

### 2. Docker 容器化

- **docker-compose.yml 配置** — 编排 postgres（pgvector）、redis、ollama、backend、frontend 五个服务。
- **Ollama 模型拉取** — `entrypoint` 中使用 `ollama serve & sleep 3 && ollama pull nomic-embed-text && ollama pull qwen2:7b && wait` 异步拉取模型，确保服务启动后自动下载所需模型。
- **健康检查** — PostgreSQL 使用 `pg_isready`、Redis 使用 `redis-cli ping`、Ollama 使用 `ollama list` 分别做健康检查，后端依赖这些服务健康后再启动。
- **Dockerfile 构建** — 后端使用多阶段构建（Maven 编译 → JRE 运行），减少镜像体积。

### 3. CORS 跨域

- **SecurityConfig** 中配置 `CorsConfigurationSource`：
  - `allowedOriginPatterns: ["*"]` 允许所有来源
  - `allowCredentials: false`（避免与通配符 Origin 冲突）
  - 支持 GET、POST、PUT、DELETE、OPTIONS 方法
  - 预检请求缓存 3600 秒

### 4. JWT 认证

- **JwtUtil** 实现 token 生成/刷新/验证/过期检查，使用 `HMAC-SHA256` 签名。
- **JwtAuthenticationFilter** 从请求头提取 token，验证后设置 `SecurityContextHolder`。
- **SecurityConfig** 配置公开端点（注册/登录/刷新 token、Swagger 文档、Actuator），其余 API 需认证。
- refresh token 有效期 = access token 的 7 倍。

### 5. 全局异常处理

- **GlobalExceptionHandler** 统一处理：
  - `ResourceNotFoundException` → 404
  - `BusinessException` → 400
  - `MethodArgumentNotValidException` → 400 + 字段级校验错误
  - `BadCredentialsException` → 401 "用户名或密码错误"
  - `MaxUploadSizeExceededException` → 400 "文件大小超过限制"
  - 其他未捕获异常 → 500 "服务器内部错误"

### 6. 文件上传

- 支持 PDF（Apache PDFBox）、DOCX（Apache POI）、纯文本文件上传与内容提取。
- 上传后自动分块（`ChunkingService.chunkByParagraph`）并生成向量嵌入索引。
- 嵌入生成失败时标记 `processed = false`，不阻塞上传流程。
- multipart 大小限制 50MB。

### 7. 向量搜索（pgvector）

- **EmbeddingService** 使用 Ollama `nomic-embed-text` 模型生成 768 维嵌入向量。
- 通过 `<=>` 余弦距离操作符进行相似度搜索，返回 top-K 结果。
- 元数据以 JSONB 格式存储（documentId、title、category、chunkIndex）。
- 删除文档时自动清理对应向量数据。

### 8. SSE 流式聊天

- **ChatService.chatStreamSse** 使用虚拟线程池异步处理：
  1. 向量搜索检索相关文档
  2. 构建 system prompt（含上下文）
  3. 通过 `chatModel.stream()` 流式输出 token
  4. 完成后发送 references（引用来源）、metadata（模型信息）、done（会话 ID）事件
  5. 持久化用户消息和完整回答
- 超时设置为 5 分钟（300000ms）。

### 9. Prompt 模板

- **TemplateService** 支持模板的 CRUD 和渲染：
  - 使用 `SimpleTemplateUtils` 进行 `{{placeholder}}` 替换
  - 默认模板 `rag-default` 用于 RAG 问答场景
  - 无模板时使用硬编码中文 system prompt 兜底
  - 未指定分类时默认分类为 "default"

### 10. 前端测试

- **Vitest** 测试框架配置完成。
- **authStore 测试** — 未认证状态、localStorage token 恢复、登录/注册成功、登出清空状态。
- **chatStore 测试** — 空状态、加载会话列表、加载历史消息、新建会话、删除会话、取消流式响应。
- **stores 测试** — documentStore、templateStore、configStore 的 CRUD 操作。
- **utils 测试** — markdown 渲染（加粗、代码块、行内代码）、相对时间格式化（刚刚/N分钟前/N小时前/N天前）、文件大小格式化（B/KB/MB）。

### 11. 后端单元测试

- **AuthControllerTest** — 注册/登录/刷新 token 接口测试（MockMvc）。
- **ChatServiceTest** — 聊天、会话管理、流式 SSE、引用来源、长标题截断。
- **DocumentServiceTest** — 文档上传、嵌入失败回退、查询、删除（含权限校验）。
- **IntegrationTest** — 完整认证流程（注册→登录→刷新）、重复用户名注册、模板 CRUD、模板渲染。使用 `@SpringBootTest` + H2 内存数据库 + `@Transactional` 自动回滚。

### 12. 数据库配置

- 开发环境使用 H2 内存数据库（`jdbc:h2:mem:testdb`），H2 Console 开启。
- 生产环境（`pg` profile）使用 PostgreSQL + pgvector，配置对应方言。
- JPA `ddl-auto: update` 自动建表。
