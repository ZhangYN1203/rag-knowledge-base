# 5.23 问题解决记录

## 项目概览

RAG 智能知识库问答系统，基于 Spring Boot 3.2.5 + Vue 3 + SiliconFlow API（DeepSeek-V3 / Qwen2.5-7B-Instruct）+ bge-m3 嵌入模型。

---

## 已解决问题

### 1. 后端启动问题

- **端口 8080 被占用** — 多次出现旧进程残留导致 `WebServerException: Port 8080 was already in use`。通过 `taskkill /PID <pid> /F` 强制终止占用进程后恢复正常启动。
- **Maven 依赖冲突** — `spring-boot-starter-webflux` 与 `spring-boot-starter-web` 共存时 Tomcat 与 Netty 冲突，通过显式排除 WebFlux 的嵌入式服务器解决（保留 WebFlux 依赖用于 WebClient，但让 Tomcat 作为主容器）。
- **Spring AI BOM 版本** — 使用 `spring-ai-bom:1.0.0` 管理 AI 相关依赖版本，确保与 Spring Boot 3.2.5 兼容。

### 2. Docker 容器化

- **docker-compose.yml 配置** — 编排 postgres（pgvector）、redis、ollama、backend、frontend 五个服务。
- **Ollama 模型拉取** — `entrypoint` 中使用 `ollama serve & sleep 3 && ollama pull nomic-embed-text && ollama pull qwen2:0.5b && wait` 异步拉取模型，确保服务启动后自动下载所需模型。
- **健康检查** — PostgreSQL 使用 `pg_isready`、Redis 使用 `redis-cli ping`、Ollama 使用 `ollama list` 分别做健康检查，后端依赖这些服务健康后再启动。
- **Dockerfile 多阶段构建** — Maven 编译 → JRE 运行，减少镜像体积。

### 3. CORS 跨域

- **SecurityConfig** 中配置 `CorsConfigurationSource`：
  - `allowedOriginPatterns: ["*"]` 允许所有来源
  - `allowCredentials: false`（避免与通配符 Origin 冲突）
  - 支持 GET、POST、PUT、DELETE、OPTIONS 方法
  - 预检请求缓存 3600 秒

### 4. JWT 认证

- **JwtUtil** 实现 token 生成/刷新/验证/过期检查，使用 `HMAC-SHA256` 签名。
- **JwtAuthenticationFilter** 从请求头提取 token，验证后设置 `SecurityContextHolder`。
- **SecurityConfig** 配置公开端点（注册/登录/刷新 token、Swagger 文档、Actuator），其余 API 需认证。
- refresh token 有效期 = access token 的 7 倍。

### 5. 全局异常处理

- **GlobalExceptionHandler** 统一处理各类异常，返回统一格式的错误响应。

### 6. 文件上传与文档解析

- 支持 PDF（Apache PDFBox）、DOCX（Apache POI）、纯文本文件上传与内容提取。
- 上传后自动分块并生成向量嵌入索引。
- 嵌入生成失败时标记 `processed = false`，不阻塞上传流程。
- multipart 大小限制 50MB。

### 7. 向量搜索

- **EmbeddingService** 使用 bge-m3 模型生成向量。
- 通过余弦余弦相似度进行 Top-K 相似度搜索。
- 支持 H2 和 PostgreSQL 双模式（H2 用 Java 计算余弦相似度，PG 用 pgvector 原生语法）。

### 8. 流式聊天与非流式切换

- 最初使用 SSE 流式输出（`SseEmitter` + 虚拟线程），但本地 Ollama 模型生成速度过慢导致用户在流式场景下体验差。
- **解决方案：** 改为非流式 POST 请求，用户看到 Loading 动画 → 全部生成后一次性展示。虽仍有等待时间，但不再出现"一个字卡 3 秒"的糟糕体验。
- 后端 SSE 流式接口 `/api/chat/stream` 保留，仅前端切换为 POST 调用。

### 9. Prompt 模板

- **TemplateService** 支持模板的 CRUD 和渲染：
  - 使用 `SimpleTemplateUtils` 进行 `{{placeholder}}` 替换
  - 默认模板 `rag-default` 用于 RAG 问答场景
  - 无模板时使用硬编码中文 system prompt 兜底

### 10. 测试

- 前端 Vitest 测试 — authStore、chatStore、documentStore、templateStore、configStore 以及 markdown/utils 工具函数。
- 后端单元测试 — Auth、Chat、Document 的 Controller/Service 测试，IntegrationTest 完整认证流程测试。

### 11. 数据库配置

- 开发环境使用 H2 文件数据库（`jdbc:h2:file:./data/ragdb`），H2 Console 开启。
- 生产环境（`pg` profile）使用 PostgreSQL + pgvector，配置对应方言。
- JPA `ddl-auto: update` 自动建表。

### 12. 模型预热问题（Ollama Manager）

- **问题：** `nomic-embed-text` 是 embedding 模型，不支持 `/api/generate` 端点，但 `OllamaModelManager` 对所有模型统一调用 warmup 导致 400 错误。
- **解决：** 跳过 embedding 模型的 generate warmup，仅 warmup 对话模型。

### 13. 本地模型速度慢

- **问题：** `qwen2:0.5b` 在 Windows CPU 上推理极慢，每个 token 需要 1-3 秒。
- **解决：** 接入云端 API（硅基流动 SiliconFlow），使用 `Qwen/Qwen2.5-7B-Instruct`（对话）+ `BAAI/bge-m3`（嵌入），响应时间从 10-60 秒降至 1-5 秒。

### 14. 前端 SSE 流式体验问题

- **问题：** 使用 SSE 流式时，因模型生成慢，页面长时间显示空白/加载状态，用户切换页面再回来才能看到完整答案。
- **解决：** 前端改为非流式 POST 请求，等待后端全部生成完毕再展示。

### 15. 对话删除失效

- **问题：** `ChatService.deleteConversation()` 缺少 `@Transactional` 注解，JPA 的 `deleteByConversationId` 无法提交事务。
- **解决：** 添加 `@Transactional` 注解。

### 16. Config 页面显示错误

- **问题：** 启动时使用 `openai` profile，但 `/config` 页面仍显示 Ollama 配置。
- **解决：** 在 `ConfigService.loadAll()` 中注入 `Environment`，根据当前 active profile 动态设置默认值。

## 技术选型变更

| 阶段 | AI 提供商 | 对话模型 | 嵌入模型 | 数据库 |
|------|-----------|---------|---------|--------|
| 初始 | Ollama（本地） | qwen2:0.5b | nomic-embed-text | PostgreSQL + PGVector |
| 当前 | SiliconFlow（云端） | Qwen/Qwen2.5-7B-Instruct | BAAI/bge-m3 | H2 File DB |
| 备选 | Ollama（本地） | qwen2:0.5b | nomic-embed-text | PostgreSQL + PGVector |

**变更原因：** 本地 CPU 推理速度极慢，云端 API 响应快、质量高、成本极低（新用户 $14 免费额度）。
