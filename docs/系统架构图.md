## 2. 交互流程

### 2.1 用户认证流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant DB as 数据库

    U->>F: 输入账号密码
    F->>B: POST /api/auth/register
    B->>DB: 检查用户名/邮箱
    DB-->>B: 不存在
    B->>DB: 保存用户（密码 BCrypt 加密）
    B->>B: 生成 JWT Token
    B-->>F: 返回 accessToken + refreshToken
    F->>F: 存储 Token 到 localStorage
    F-->>U: 跳转到聊天页面
```

### 2.2 文档上传与向量化

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant AI as AI API
    participant DB as 数据库

    U->>F: 选择文件上传
    F->>B: POST /api/documents (multipart)
    B->>B: 解析文件内容 (PDF/DOCX/TXT)
    B->>B: 文本分块 (Chunking)
    loop 每个文本块
        B->>AI: 请求嵌入向量 (bge-m3)
        AI-->>B: 返回向量
        B->>DB: 存储向量到 vector_store 表
    end
    B->>DB: 保存文档元数据
    B-->>F: 返回上传成功
    F-->>U: 刷新文档列表
```

### 2.3 智能问答流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant AI as AI API
    participant DB as 数据库

    U->>F: 输入问题
    F->>B: POST /api/chat

    Note over B: 1. 向量检索
    B->>AI: 问题向量化 (bge-m3)
    AI-->>B: 返回向量
    B->>DB: 向量相似度搜索 (Top-5)
    DB-->>B: 返回相关文档片段

    Note over B: 2. 构建 Prompt
    B->>B: 系统模板 + 上下文注入

    Note over B: 3. 调用 LLM 生成回答
    B->>AI: Chat (Qwen2.5-7B-Instruct)
    AI-->>B: 完整回答

    B->>DB: 持久化对话记录
    B-->>F: 返回答案 + 引用来源
    F->>F: Markdown 渲染展示
    F-->>U: 显示完整回答
```

## 4. 组件依赖关系

| 组件 | 依赖 | 说明 |
|------|------|------|
| Spring Boot Backend | SiliconFlow API、H2/PostgreSQL | 核心业务逻辑 |
| Vue 3 Frontend | Backend API | 用户界面 |
| SiliconFlow API | - | 外部 AI 模型服务（对话+嵌入） |
| H2 / PostgreSQL | - | 持久化数据 + 向量检索 |

> **注：** 生产环境使用 H2 文件数据库 + SiliconFlow API，无需额外部署 PostgreSQL 或 Ollama。
> 开发环境支持切换至本地 Ollama（仅需更改 Spring profile）。
