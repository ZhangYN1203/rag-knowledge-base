## 2. 交互流程

### 2.1 用户认证流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant DB as 数据库

    U->>F: 输入账号密码
    F->>B: POST /api/auth/register
    B->>DB: 检查用户名/邮箱
    DB-->>B: 不存在
    B->>DB: 保存用户（密码 BCrypt 加密）
    B->>B: 生成 JWT Token
    B-->>F: 返回 accessToken + refreshToken
    F->>F: 存储 Token 到 localStorage
    F-->>U: 跳转到聊天页面

    Note over U,B: 后续请求自动携带 Authorization: Bearer <token>
```

### 2.2 文档上传与向量化

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant O as Ollama
    participant DB as 数据库

    U->>F: 选择文件上传
    F->>B: POST /api/documents (multipart)
    B->>B: 解析文件内容 (PDF/DOCX/TXT)
    B->>B: 文本分块 (Chunking)
    loop 每个文本块
        B->>O: 请求嵌入向量 (nomic-embed-text)
        O-->>B: 返回 768 维向量
        B->>DB: 存储向量到 vector_store 表
    end
    B->>DB: 保存文档元数据到 documents 表
    B-->>F: 返回上传成功
    F-->>U: 刷新文档列表
```

### 2.3 智能问答流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant O as Ollama
    participant DB as 数据库

    U->>F: 输入问题
    F->>B: GET /api/chat/stream?message=...
    Note over B: 建立 SSE 连接

    B->>O: 问题向量化 (nomic-embed-text)
    O-->>B: 返回向量
    B->>DB: 向量相似度搜索 (Top-5)
    DB-->>B: 返回相关文档片段

    B->>B: 构建 Prompt (模板 + 上下文)
    B->>O: Chat Stream (qwen2:0.5b)

    loop SSE 流式推送
        O-->>B: 逐 token 生成
        B-->>F: event: token / data: "文本"
        F->>F: ReadableStream 读取
        F->>F: Markdown 渲染展示
    end

    B-->>F: event: references (引用来源)
    B-->>F: event: done (会话 ID)
    B->>DB: 持久化对话记录
    Note over F: 关闭 SSE 连接
    F-->>U: 显示完整回答 + 引用
```

## 4. 组件依赖关系

| 组件 | 依赖 | 说明 |
|------|------|------|
| Spring Boot Backend | PostgreSQL, Redis, Ollama | 核心业务逻辑 |
| Vue 3 Frontend | Backend API | 用户界面 |
| Ollama | - | 独立的 AI 模型服务 |
| PostgreSQL | - | 持久化数据 + 向量检索 |

> 注：开发环境下使用 H2 文件数据库替代 PostgreSQL，使用本地 Ollama 服务替代容器化 Ollama。
